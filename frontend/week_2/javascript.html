JavaScript

JavaScript is a dynamically typed, functional, objected oriented and imperative programming language. It was born out of making pages interactive in the early days of the internet. Prior to JavaScript, pages only had rudimentary interactivity such as clicking on links redirecting the user. With modern JavaScript and browser APIs such as Canvas, WebGL and Web Audio, complex applications can be made for the browser.

JavaScript as a programming language

JavaScript is a C-like language, which is a family of languages based off C. Examples include C++, Java and C#. The syntax for JavaScript is largely the same as the aforementioned languages, with curly braces for defining scope, dots for accessing object members and semicolon line endings.

Unlike its relative though, it is a dynamically typed language, which is often JIT compiled (not compiled upfront). This means you do not type any variables, so the programmer must understand what the intended type for a variable is by looking at the code. Variables can be assigned to any type at any point.

JavaScript also has automatic lifetime control for objects in memory. In C/C++, every single piece of memory that is allocated on the heap must be managed manually - using malloc/free or new/delete. In JavaScript this is not necessary, nor possible. You don't have to worry too much about what this means, the only implication is that you can't delete things out of memory as the programmer. The browser will do this for you at some point.

JavaScript's object model is also different to its C-like cousins. JavaScript uses prototypes to implement OO, which is quite different to most programming languages. Most programming languages use class based object orientation. The fact that JavaScript is dynamically typed, and that it uses prototypical inheritance means that there are quirks to using OO in JavaScript compared to other languages. We can get into this later down the line.

JavaScript has functional programming language features. The first and foremost feature is the fact that functions are first class objects. This means that functions can be assigned to variables like any other object, and also manipulated as objects.

An important concept in functional programming is the use of callbacks. This is simply a function that is passed to another function to be used at a later point. This can be chained as deeply as the programmer wishes.

Functions are defined using the `function` keyword, or arrow functions. `function` functions can either be named or anonymous. Regular functions and arrow functions differ mainly in how the `this` keyword is bound - arrow functions bind `this` lexically (looks at the surrounding scope) and `function` uses the calling context as `this`, which basically means that for regular functions `this` can mean different things depending on where it is called, whereas arrow functions always have the same meaning for `this`

How JavaScript is included in a web page

JavaScript is included in a HTML page in three ways:

  • Directly written inside a script tag
  • A script tag with a `src` attribute pointing to a remote JS tag (most common)
  • Written in an attribute of an HTML element (e.g. on an `onclick` attribute)

Web API ('Web Page' JavaScript)

There is a specific set of interfaces just for the browser. We've used `console`, `document` and `window` so far. They will be the most used interfaces for interfacing with the web page or the browser.

Just like any API, you will learn this organically through searching the internet and from reading web code. 99% of the time you will be using the following:

  • document (interface to the DOM)
  • window (interface to the browser window)
  • console (interface for debugger)
  • Event API
  • Promise API (Asynchronous programming API)
  • Fetch API (For asynchronous fetching of remote files)
